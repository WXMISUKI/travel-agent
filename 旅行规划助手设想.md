利用 Minimax 的 API，构建一个能完成特定任务的 Agent。
第一步：环境准备
去 Minimax 官网注册，获取 api_key。
安装必要的 SDK（通常 Minimax 会提供 Python SDK）。
第二步：Demo 架构设计（简单版）
最经典的入门 Agent Demo 是 “旅行规划助手” 或 “研究分析师”。
核心逻辑：
接收用户指令（例如：“帮我规划一个去上海的三天两夜旅行”）。
调用大模型（Minimax）：让它分析需求，并决定是否需要调用工具。
工具调用（Function Calling）：
如果模型觉得需要查天气，它会输出一个函数调用指令（如 get_weather(location="上海")）。
你的代码需要捕获这个指令，执行真实的查询（或者模拟），然后把结果喂回给模型。
最终输出：模型整合信息，生成最终回答。
第三步：利用 Gemini/ChatGPT 辅助开发
利用 AI生成代码框架：
Prompt: “请帮我写一个 Python 脚本，使用 Minimax 的 API 实现一个简单的 Agent。要求能接收用户输入，并流式输出回答。请使用官方 SDK。”
实现 Function Calling：
Prompt: “如何在 Minimax API 中实现工具调用（Function Calling）？请给出一个例子，包含一个查询天气的工具函数。”
调试与修改：
如果报错，把错误信息丢给 Gemini：“这段代码报错了 [错误信息]，请帮我修复。”
