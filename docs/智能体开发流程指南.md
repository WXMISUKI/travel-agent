## 二、阶段详解

### 阶段 1：需求分析与项目确定

#### 1.1 核心目标

明确智能体的**边界、价值、用户场景**，把模糊需求转化为可落地的量化指标。

#### 1.2 关键步骤

**Step 1：确定服务领域**

| 问题 | 示例答案 |
|------|----------|
| 智能体服务什么领域？ | 旅行规划 |
| 目标用户是谁？ | 计划出行的人群 |
| 解决什么问题？ | 查询天气、火车票、景点信息繁琐 |
| 核心价值是什么？ | 一站式获取旅行信息 |

**Step 2：明确能力边界**

列出智能体**能做什么**和**不能做什么**：

```
能做的：
✅ 查询目的地天气预报
✅ 查询火车票余票信息
✅ 推荐热门景点和美食
✅ 提供旅行建议和攻略

不能做的：
❌ 预订酒店、机票、火车票（只能查询）
❌ 代替用户下单支付
❌ 提供医疗、法律等专业建议
```

**Step 3：定义用户场景**

| 场景 | 用户输入示例 | 期望输出 |
|------|-------------|----------|
| 查天气 | "后天杭州天气怎么样" | 杭州后天天气预报 |
| 查火车票 | "明天北京到上海的高铁票" | 余票信息列表 |
| 查景点 | "上海有什么好玩的地方" | 景点推荐列表 |
| 规划行程 | "帮我规划一个去厦门的三天行程" | 完整旅行计划 |

#### 1.3 输出产物

- 📄 **需求文档**：明确功能列表、用户场景、能力边界
- 📊 **MVP定义**：最小可行产品包含的核心功能

### 阶段 2：技术选型与方案制定

#### 2.1 核心目标

选择合适的技术栈和方案，确保可行性、成本可控、易于维护。

#### 2.2 技术选型决策

**LLM选择：**

| 选项 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| OpenAI GPT-4 | 能力强、生态好 | 国内访问受限、价格高 | 追求极致效果 |
| MiniMax M2.5 | 中文优化、国内可用 | 能力略逊GPT-4 | 国内项目首选 |
| 阿里云DashScope | 稳定、易集成 | 依赖阿里云 | 国内企业项目 |
| 开源模型（Qwen等） | 可私有部署 | 效果不稳定 | 数据敏感场景 |

**本项目选择：MiniMax M2.5（阿里云DashScope）**

**Agent框架选择：**

| 框架 | 特点 | 学习曲线 |
|------|------|----------|
| LangChain | 生态丰富文档全 | 中等 |
| LangGraph | 结构化、可视化 | 较陡 |
| AutoGen | 多Agent协作 | 中等 |
| 自研 | 完全可控 | 较陡 |

**本项目选择：LangChain + LangGraph**

**数据源策略：**

```
数据源优先级：
1. 免费开源API（如Open-Meteo天气）
2. 官方API（如12306 MCP）
3. 第三方API（如百度搜索降级）
```

#### 2.3 输出产物

- 📋 **技术选型报告**：说明为什么选择这些技术
- 💰 **成本估算**：API调用费用、服务器成本等

#### 3.3 核心模块设计

**Agent主类 (TravelAgent)**

```python
class TravelAgent:
    """旅行规划Agent"""
    
    def __init__(self):
        # 初始化LLM客户端
        self.llm = ChatOpenAI(...)
        # 加载可用工具
        self.tools = get_all_tools()
    
    def run(self, user_input: str) -> str:
        # 1. 意图识别 - 判断用户想要什么
        intent = self._parse_intent(user_input)
        
        # 2. 实体提取 - 提取关键信息（地点、时间等）
        entities = self._extract_entities(user_input)
        
        # 3. 执行计划生成 - 规划如何完成
        plan = self.create_execution_plan(intent, entities)
        
        # 4. 工具执行 - 调用API获取数据
        results = self.execute_plan(plan)
        
        # 5. 响应生成 - 组织语言回复用户
        return self._make_response(results)
```

**智能计划生成器 (SmartPlanner)**

```python
class SmartPlanner:
    """智能执行计划生成器"""

        IntentType.WEATHER: [r"天气", r"温度", ...],
        IntentType.ATTRACTIONS: [r"景点", r"美食", ...],
        # ...
    }
    
    def generate_plan(self, query: str, intent, entities) -> List[ExecutionStep]:
        """根据意图和实体生成执行步骤"""
        
        if intent == IntentType.WEATHER:
            # 需要：日期 + 目的地
            return [
                Step(1, "parse_date", {"date": date_text}),
                Step(2, "get_weather", {"city": destination})
            ]
        
        elif intent == IntentType.TRIP_PLAN:
            # 需要：完整旅行规划流程
            return [
                Step(1, "parse_date", {...}),
                Step(2, "get_weather", {...}),
                Step(3, "search_attractions", {...}),
                Step(4, "web_search", {...})
            ]
```

# src/agent/tools.py
import json

def get_weather(city: str) -> str:
    """查询城市天气
    
    参数:
        city: 城市名称，如"北京"、"杭州"
    
    返回:
        JSON格式的天气数据

    # 调用天气API
    weather_data = call_weather_api(city)
    # json的dumps函数是为了将数据转换为JSON字符串

**Step 4：实现意图识别**

```python
# 方式1：规则匹配（简单高效）
INTENT_PATTERNS = {
    "train_tickets": [r"火车票", r"高铁", r"动车"],
    "weather": [r"天气", r"温度", r"下雨"],
    "attractions": [r"景点", r"好玩", r"美食"]
}

def parse_intent(query: str) -> str:
    for intent, patterns in INTENT_PATTERNS.items():
        for pattern in patterns:
            if re.search(pattern, query):
                return intent
    return "general"

# 方式2：LLM辅助（更准确）
def parse_intent_with_llm(query: str) -> str:
    prompt = f"""用户问题: {query}
判断用户意图，只能选择: train_tickets, weather, attractions, general
直接输出JSON: {{"intent": "..."}}"""
    
    result = llm.chat(prompt, query)
    return json.loads(result)["intent"]
```

**Step 5：实现实体提取**

```python
def extract_entities(query: str) -> dict:
    """提取关键实体
    
    返回:
    {
        "origin": "北京",
        "destination": "上海", 
        "date_text": "明天",
        "train_type": "高铁"
    }
    """
    entities = {}
    
    # 提取地点
    entities["destination"] = extract_city(query)
    
    # 提取日期
    entities["date_text"] = extract_date(query)
    
    # 提取火车类型
    if "高铁" in query:
        entities["train_type"] = "G"
    elif "动车" in query:
        entities["train_type"] = "D"
    
    return entities
```

**Step 6：实现工具执行器**

```python
def execute_tool(tool_name: str, params: dict) -> str:
    """执行工具函数"""
    
    if tool_name not in AVAILABLE_TOOLS:
        return json.dumps({"error": f"未知工具: {tool_name}"})
    
    tool_func = AVAILABLE_TOOLS[tool_name]["function"]
    
    try:
        result = tool_func(**params)
        return result
    except Exception as e:
        return json.dumps({"error": str(e)})
```

#### 5.3 常见问题与调优

**问题1：意图识别不准确**

```
症状：用户问天气，识别成景点查询
原因：规则模式不够全面
解决：
1. 增加更多匹配模式
2. 引入LLM辅助判断
3. 添加意图确认环节
```

**问题2：实体提取遗漏**

```
症状：目的地没有提取出来
原因：城市名不在词库中
解决：
1. 扩充城市名映射
2. 使用更强大的NER模型
3. LLM辅助提取
```

**问题3：工具调用失败**

```
症状：API调用超时或返回错误
原因：网络问题、API不稳定
解决：
1. 添加超时设置
2. 实现重试机制
3. 添加降级方案
```

**问题4：响应质量不佳**

```
症状：回复不够自然、不够有用
原因：Prompt不够好、缺少上下文
解决：
1. 优化Prompt模板
2. 添加更多上下文信息
3. 调整LLM参数（temperature等）
```

```python
def safe_execute_tool(tool_name, params):
    try:
        return execute_tool(tool_name, params)
    except TimeoutError:
        logger.warning(f"工具 {tool_name} 超时")
        return get_fallback_result(tool_name)
    except Exception as e:
        logger.error(f"工具 {tool_name} 异常: {e}")
        return get_fallback_result(tool_name)
```
