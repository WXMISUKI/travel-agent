# 旅行规划智能体说明文档

## 一、项目概述

### 1.1 项目简介

旅行规划智能体（Travel Agent）是一个基于大语言模型（LLM）的智能旅行助手，能够理解用户的自然语言查询，自动规划执行步骤，并调用各类旅行相关API获取信息，最终生成友好的回复。

**核心能力：**
- 🌤️ 天气查询 - 查询目的地15天天气预报
- 🚄 火车票查询 - 查询12306列车余票信息
- 🎯 景点推荐 - 推荐热门景点和网红打卡地
- 🔍 智能问答 - 回答各类旅行相关问题
- 📅 行程规划 - 自动生成旅行计划

### 2.2 核心组件

#### 2.2.1 TravelAgent（主代理类）

**位置：** `src/agent/graph.py`

**职责：**
- 协调整个Agent的运行流程
- 管理意图识别、实体提取、执行计划、工具调用
- 处理降级逻辑和响应生成

**核心方法：**

```python
class TravelAgent:
    def run(self, user_input: str, history: List[Dict] = None) -> str:
        """主运行方法"""
        # 1. 解析意图
        intent = self._parse_intent(user_input)
        # 2. 提取实体
        entities = self._extract_entities(user_input)
        # 3. 生成执行计划
        plan = self.create_execution_plan(user_input, intent, entities)
        # 4. 执行计划
        results = self.execute_plan(plan)
        # 5. 生成响应
        return self._make_response(user_input, intent, results)
```

#### 2.2.2 SmartPlanner（智能计划生成器）

**位置：** `src/agent/smart_planner.py`

**职责：**
- 分析用户查询，确定需要调用的工具链
- 自动推断缺失参数
- 生成多步骤执行计划

**支持的意图类型：**

| 意图类型 | 说明 | 匹配关键词 |
|----------|------|------------|
| TRAIN_TICKETS | 火车票查询 | 火车票、高铁、动车、坐火车 |
| WEATHER | 天气查询 | 天气、温度、下雨、晴天 |
| ATTRACTIONS | 景点查询 | 景点、美食、好玩、网红 |
| TRIP_PLAN | 旅行规划 | 规划、行程、旅游攻略 |
| CAPABILITY | 能力查询 | 你能做什么、有哪些功能 |

#### 2.2.3 Tools（工具层）

**位置：** `src/agent/tools.py`

**可用工具：**

| 工具名称 | 功能 | 数据来源 |
|----------|------|----------|
| `get_weather` | 查询15天天气预报 | apihz.cn / Open-Meteo |
| `get_train_tickets` | 查询火车票余票 | 12306 MCP |
| `get_station_by_city` | 查询城市火车站 | 12306 MCP |
| `search_attractions` | 搜索景点美食 | 百度搜索 |
| `web_search` | 通用搜索（降级用） | 百度搜索 |

## 三、工作流程详解

### 3.1 完整工作流程

```
用户输入： "我想去杭州玩三天，帮我查一下天气和景点"

┌─────────────────────────────────────────────────────────────────┐
│  Step 1: 意图识别                                              │
│  输入：用户原始查询                                              │
│  输出：INTENT_TYPE = TRIP_PLAN                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 2: 实体提取                                               │
│  输入：用户查询 + 当前时间                                       │
│  输出：destination="杭州", date_text="三天", keyword="景点"    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 3: 执行计划生成                                           │
│  输入：意图 + 实体                                               │
│  输出：                                                          │
│  [1] parse_date("三天")                                         │
│  [2] get_weather("杭州")                                        │
│  [3] search_attractions("杭州", "景点")                         │
│  [4] web_search("杭州旅游攻略")                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 4: 工具执行                                               │
│  依次执行计划中的工具，收集结果                                   │
│  - 如果主工具失败，启动降级搜索                                   │
│  - 记录每一步的执行状态                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 5: 响应生成                                               │
│  输入：用户查询 + 所有工具结果 + 时间上下文                       │
│  输出：自然语言回复                                              │
│  "您好！根据您的需求，我帮您查询到..."                           │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 意图识别流程

**实现方式：** 规则匹配 + LLM辅助

1. **规则匹配阶段：** 使用正则表达式匹配常见模式
2. **LLM确认阶段：** 对于复杂查询，使用LLM进行意图确认

```python
# 意图模式匹配示例
INTENT_PATTERNS = {
    IntentType.TRAIN_TICKETS: [
        r"火车票", r"高铁", r"动车", r"普快", r"坐火车",
        r"到.*去", r"去.*要"
    ],
    IntentType.WEATHER: [
        r"天气", r"温度", r"下雨", r"晴天", r"冷不冷"
    ],
    # ...
}
```

### 3.3 实体提取流程

**提取的实体类型：**
- `origin`：出发地
- `destination`：目的地
- `date_text`：日期描述
- `train_type`：火车类型（G/D/K）
- `keyword`：搜索关键词
- `budget`：预算

**处理逻辑：**
1. 首先使用正则规则提取
2. 对复杂描述使用LLM进行提取
3. 结合时间上下文进行标准化

### 3.4 降级策略

当主工具调用失败时，自动触发降级：

```python
def _call_with_fallback(self, tool_name: str, params: dict, fallback):
    """调用工具，失败时降级"""
    try:
        result = execute_tool(tool_name, params)
        # 检查结果是否有效
        if "error" in result:
            return fallback()  # 降级到备用方案
        return result
    except Exception:
        return fallback()
```

**降级链路：**
- `get_weather` → `web_search` (百度搜索天气)
- `get_train_tickets` → `web_search` (百度搜索火车票)
- `search_attractions` → `web_search` (百度搜索景点)

## 八、常见问题

### 8.1 Q: 如何获取MiniMax API Key？

A: 访问[阿里云DashScope](https://dashscope.console.aliyun.com/)注册并创建API Key。

### 8.2 Q: 火车票查询需要启动什么服务？

A: 需要启动 `mcps/12306-mcp` 项目中的MCP服务。

### 8.3 Q: 天气查询支持哪些城市？

A: 当前支持约40个主要城市，包括北京、上海、广州、深圳、杭州、成都等。

### 8.4 Q: 如何查看详细的执行过程？

A: 查看 `logs/travel-agent.log` 日志文件，或使用 `/audit/events` API查询审计记录。
