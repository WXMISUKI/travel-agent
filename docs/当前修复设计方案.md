## 二、当前存在的缺陷（2026-03-14 新增严重BUG）

### 2.0 严重BUG：无限递归导致栈溢出（紧急修复）

#### 2.0.1 问题描述

**现象**：测试天气查询时，程序一直调用接口且报错，强制关闭后端才停止。

**日志证据**：
```
2026-03-14 11:34:48.415 | ERROR | src.agent.tools:execute_tool_with_retry:816 - 工具 get_weather 执行异常: maximum recursion depth exceeded
2026-03-14 11:34:48.917 | ERROR | src.agent.tools:execute_tool_with_retry:816 - 工具 get_weather 执行异常: maximum recursion depth exceeded
...（无限循环）
```

#### 2.0.2 根本原因

代码结构问题导致无限递归：

```python
# tools.py 第785行 - execute_tool_with_retry 函数内部
def execute_tool_with_retry(tool_name: str, arguments: dict) -> str:
    for attempt in range(max_retries + 1):
        try:
            # 调用原始 execute_tool
            result = execute_tool(tool_name, current_params)  # ← 调用 execute_tool
            ...
        except Exception as e:
            ...

# tools.py 第854行 - execute_tool 函数
def execute_tool(tool_name: str, arguments: dict) -> str:
    """执行工具 - 带重试机制"""
    return execute_tool_with_retry(tool_name, arguments)  # ← 又调用 execute_tool_with_retry
```

**问题**：`execute_tool_with_retry` 内部调用 `execute_tool`，而 `execute_tool` 又调用 `execute_tool_with_retry`，形成循环递归调用，导致栈溢出。

#### 2.0.3 修复方案

**方案A：直接执行函数（推荐）**

```python
def execute_tool_with_retry(tool_name: str, arguments: dict) -> str:
    """带重试机制的工具执行"""
    # 直接从 AVAILABLE_TOOLS 获取函数执行，不再调用 execute_tool
    if tool_name not in AVAILABLE_TOOLS:
        return json.dumps({"error": f"未知工具: {tool_name}"}, ensure_ascii=False)
    
    func = AVAILABLE_TOOLS[tool_name]["function"]
    
    for attempt in range(max_retries + 1):
        try:
            # 直接调用函数
            if arguments:
                result = func(**arguments)
            else:
                result = func()
            result = result if isinstance(result, str) else str(result)
            
            # 检查结果是否为错误
            if not _is_error_result(result):
                return result
            ...
```

**方案B：引入中间层**

```python
def _execute_tool_raw(tool_name: str, arguments: dict) -> str:
    """原始工具执行（内部使用）"""
    if tool_name in AVAILABLE_TOOLS:
        func = AVAILABLE_TOOLS[tool_name]["function"]
        try:
            if arguments:
                result = func(**arguments)
            else:
                result = func()
            return result if isinstance(result, str) else str(result)
        except Exception as e:
            return json.dumps({"error": str(e)}, ensure_ascii=False)
    return json.dumps({"error": f"未知工具: {tool_name}"}, ensure_ascii=False)

def execute_tool_with_retry(tool_name: str, arguments: dict) -> str:
    """带重试机制的工具执行"""
    for attempt in range(max_retries + 1):
        result = _execute_tool_raw(tool_name, arguments)
        if not _is_error_result(result):
            return result
        # 重试逻辑...
    return result

def execute_tool(tool_name: str, arguments: dict) -> str:
    """执行工具 - 带重试机制"""
    return execute_tool_with_retry(tool_name, arguments)
```

### 2.1 工具层面

| 缺陷 | 描述 | 影响 |
|------|------|------|
| **机票接口未授权** | get_flight_tickets 返回需要授权错误 | 无法查询机票 |
| **周边景点关键词不精确** | 搜索"景点"返回的结果质量一般 | 用户体验一般 |
| **火车站名称解析** | 部分小站名称解析不准确 | 火车票查询可能失败 |

### 2.2 工作流层面

| 缺陷 | 描述 | 影响 |
|------|------|------|
| **缺乏重试机制** | 工具失败后直接降级，无重试 | 降级过于激进 |
| **参数修复不完善** | 错误参数修复逻辑简单 | 纠错能力弱 |
| **无记忆机制** | 每次对话都是全新会话 | 无法记住用户偏好 |

### 2.3 架构层面

| 缺陷 | 描述 | 影响 |
|------|------|------|
| **未使用 LangGraph 状态管理** | 虽然引入了 LangGraph，但状态管理仍是自定义 | 架构不够标准 |
| **工具注册分散** | 工具定义在多个文件 | 维护困难 |
| **缺乏统一的错误处理** | 各工具错误处理不一致 | 调试困难 |

### 2.4 体验层面

| 缺陷 | 描述 | 影响 |
|------|------|------|
| **多轮对话支持弱** | 无法记住上下文 | 交互不自然 |
| **用户偏好未记住** | 常用出发地每次需要重新输入 | 效率低 |
| **推荐不够智能** | 缺乏主动推荐能力 | 功能单一 |
