# 旅行规划智能体 - 技术栈与功能说明

## 项目简介

基于 **LangGraph** 和 **MiniMax 大模型** 构建的智能旅行规划助手，实现从自然语言理解到复杂任务编排的端到端解决方案。项目采用企业级架构设计，具备智能规划、自动降级、时间上下文管理和完整审计追踪能力。

## 核心功能

### 1. 智能对话理解
- **意图识别**：支持 7 类意图（火车票、天气、景点、交通、住宿、能力查询、旅行规划）
- **实体提取**：自动提取出发地、目的地、日期、车次类型等关键信息
- **上下文管理**：多轮对话的上下文理解和状态维护

### 2. 智能执行规划
- **DAG 任务编排**：有向无环图管理工具依赖关系
- **自动参数推断**：智能补全缺失参数
- **执行计划生成**：基于意图生成最优执行步骤

### 3. 时间上下文管理
- **自然语言日期解析**：支持"明天"、"下周五"等表达
- **相对时间计算**：自动计算相对日期
- **时区处理**：中国时区时间管理
- **日期标准化**：统一的日期格式输出

**支持的表达：**
- 相对日期：今天、明天、后天、大后天
- 星期表达：周一、周二、下周三、这周末
- 周期表达：本周、下周、下个月

### 4. 多数据源集成
| 数据源 | 类型 | 用途 | 状态 |
|--------|------|------|------|
| Open-Meteo | 开源 API | 天气查询 | ✅ 已实现 |
| 12306 MCP | MCP 协议 | 火车票查询 | ✅ 已实现 |
| 百度搜索 | 搜索 API | 降级搜索 | ✅ 已实现 |
| 航班数据 | 预留接口 | 航班查询 | 🔜 待开发 |
| 附近信息 | 预留接口 | 周边查询 | 🔜 待开发 |

### 5. 智能降级系统
- **参数修复**：自动修复错误参数（"北京县"→"北京市"）
- **工具降级**：主工具失败时切换备用工具
- **搜索降级**：专业数据源失败时使用通用搜索
- **友好提示**：降级失败时的错误提示

**降级策略示例：**
```
火车票查询 → 修复站名 → 搜索"北京到上海火车票" → 返回结果
```

### 6. 企业级审计系统
- **会话追踪**：完整的用户会话记录
- **执行追踪**：工具调用链和执行时间
- **性能监控**：响应时间、成功率等指标
- **错误分析**：详细的错误日志和降级记录

**审计事件：**
- 用户查询、意图识别、实体提取
- 计划创建、工具调用、工具结果
- 降级、响应生成

### 7. 流式响应输出
- **实时推送**：基于 SSE 的实时事件流
- **状态展示**：思考过程、执行计划、步骤状态
- **用户体验**：分步骤展示，提升交互体验

**响应事件流：**
```json
{"type": "start", "session_id": "abc123"}
{"type": "thinking", "content": ["🧠 分析需求..."]}
{"type": "todo", "items": ["⬜ [1] 解析日期"]}
{"type": "step_status", "step_id": "1", "status": "completed"}
{"type": "content", "content": "为您查到..."}
{"type": "done"}
```

## 核心设计模式

### 1. 智能规划模式（Smart Planning Pattern）
- 基于意图识别的执行计划生成
- 支持工具依赖和条件分支
- 自动参数推断和补全

### 2. 降级策略模式（Fallback Pattern）
- 多级降级机制
- 自动错误检测和恢复
- 参数智能修复

### 3. 时间上下文模式（Time Context Pattern）
- 会话级时间状态管理
- 自然语言日期解析
- 相对时间计算

### 4. 审计追踪模式（Audit Trail Pattern）
- 结构化事件日志
- 执行链追踪
- 性能指标收集

## 技术难点与解决方案

### 难点 1：复杂的依赖关系管理
**解决方案：**
- DAG 表示依赖关系
- 拓扑排序确定执行顺序
- 参数占位符解析

### 难点 2：自然语言日期解析
**解决方案：**
- 时间上下文管理器
- 支持相对日期和星期表达
- 时区处理和标准化

### 难点 3：数据源不稳定
**解决方案：**
- 多级降级策略
- 参数智能修复
- 自动错误恢复

### 难点 4：性能监控和调试
**解决方案：**
- 企业级审计日志系统
- 结构化事件记录
- 执行链追踪

### 难点 5：实时用户体验
**解决方案：**
- SSE 流式响应
- 实时推送执行状态
- 异步工具调用
