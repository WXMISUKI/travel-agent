# 旅行规划智能体 - 项目技术说明文档

## 项目概述

这是一个基于 **LangGraph** 框架构建的智能旅行规划助手，采用企业级架构设计，集成了大模型、多数据源、智能规划和审计系统。项目实现了从自然语言理解到复杂任务规划的端到端解决方案，具备智能降级、时间上下文管理和完整的审计追踪能力。

**项目定位：** 垂域智能体（Domain-Specific Agent）的最佳实践案例
**开发周期：** 持续迭代，目前版本 v2.0.0
**技术难度：** ⭐⭐⭐⭐⭐（企业级智能体系统）

### 核心设计模式

1. **智能规划模式（Smart Planning Pattern）**
   - 基于意图识别的执行计划生成
   - 支持工具依赖和条件分支
   - 自动参数推断和补全

2. **降级策略模式（Fallback Pattern）**
   - 多级降级机制
   - 自动错误检测和恢复
   - 参数智能修复

3. **时间上下文模式（Time Context Pattern）**
   - 会话级时间状态管理
   - 自然语言日期解析
   - 相对时间计算

4. **审计追踪模式（Audit Trail Pattern）**
   - 结构化事件日志
   - 执行链追踪
   - 性能指标收集

## 核心功能

### 1. 智能对话理解

**功能描述：** 基于大模型的自然语言理解，支持复杂旅行需求的解析

**技术实现：**
- 意图分类：支持火车票、天气、景点、交通、住宿等 7 类意图
- 实体提取：提取出发地、目的地、日期、车次类型等关键信息
- 上下文维护：多轮对话的上下文理解和状态管理

**核心代码：**
```python
class SmartPlanner:
    """智能执行计划生成器"""
    INTENT_PATTERNS = {
        IntentType.TRAIN_TICKETS: [r"火车票", r"高铁", r"动车"],
        IntentType.WEATHER: [r"天气", r"温度", r"下雨"],
        IntentType.ATTRACTIONS: [r"景点", r"美食", r"推荐"],
        # ... 更多意图模式
    }
```

### 2. 智能执行规划

**功能描述：** 自动生成最优执行计划，支持工具依赖和并行执行

**技术实现：**
- DAG（有向无环图）任务编排
- 工具依赖关系管理
- 参数占位符解析
- 执行步骤优先级排序

**核心代码：**
```python
@dataclass
class ExecutionStep:
    """执行步骤"""
    id: int
    tool: str
    params: Dict[str, Any]
    purpose: str
    depends_on: Optional[int] = None  # 依赖关系
    fallback: Optional[Dict] = None   # 降级配置
```

### 3. 时间上下文管理

**功能描述：** 智能解析自然语言日期表达，支持相对时间计算

**技术实现：**
- 中国时区时间管理
- 相对日期解析（"明天"、"下周五"）
- 星期几转换
- 日期格式标准化

**支持的表达：**
- 相对日期：今天、明天、后天、大后天
- 星期表达：周一、周二、下周三、这周末
- 相对周期：本周、下周、下个月

**核心代码：**
```python
class TimeContext:
    """时间上下文管理"""
    def parse_natural_date(self, text: str) -> Optional[str]:
        """解析自然语言日期"""
        # 支持多种日期表达
        date_patterns = {
            "今天": 0, "明天": 1, "后天": 2,
            "下周": lambda x: (x - today) % 7 + 7
        }
```

### 5. 智能降级系统

**功能描述：** 多级降级策略，确保服务可用性

**降级层级：**
1. **参数修复**：自动修复错误参数（如"北京县"→"北京市"）
2. **工具降级**：主工具失败时切换备用工具
3. **搜索降级**：专业数据源失败时使用通用搜索
4. **错误提示**：降级失败时友好提示

**降级策略示例：**
```python
# 火车票查询降级策略
train_ticket → 修复站名 → 搜索"北京到上海火车票" → 返回搜索结果
```

## 技术亮点

### 1. LangGraph 状态机工作流

**技术难点：** 实现复杂的多步骤任务编排

**解决方案：**
- 使用 LangGraph 框架构建状态机
- 定义节点和边，实现灵活的工作流
- 支持条件分支和循环

**代码实现：**
```python
from langgraph.graph import StateGraph, END

workflow = StateGraph(AgentState)

# 添加节点
workflow.add_node("parse_intent", parse_intent_node)
workflow.add_node("create_plan", create_plan_node)
workflow.add_node("execute_tools", execute_tools_node)
workflow.add_node("generate_response", generate_response_node)

# 添加边
workflow.add_edge("parse_intent", "create_plan")
workflow.add_conditional_edges(
    "create_plan",
    should_execute_tools,
    {"execute": "execute_tools", "skip": "generate_response"}
)
```

### 2. MCP 协议集成

**技术难点：** 实现与 12306 服务的标准化通信

**解决方案：**
- 遵循 MCP（Model Context Protocol）协议
- 封装 MCP 客户端，处理协议细节
- 实现服务发现和能力协商

### 3. 智能参数修复

**技术难点：** 自动检测和修复错误参数

**解决方案：**
- 分析错误返回信息
- 提取错误原因和修复建议
- 自动应用修复策略

**代码实现：**
```python
def _try_fix_tool_params(tool_name: str, params: Dict, error_result: Dict):
    """尝试修复工具参数"""
    error_msg = error_result.get("error", "")
    
    if tool_name == "get_train_tickets":
        # 尝试从错误信息提取有效站名
        station_match = re.search(r'([^\s,，、。]{2,6}站)', error_msg)
        
        # 尝试去掉县等后缀
        for key in ["from_station", "to_station"]:
            if key in fixed and "县" in fixed[key]:
                fixed[key] = fixed[key].replace("县", "市")
```

### 4. 并发工具调用优化

**技术难点：** 优化无依赖工具的执行效率

**解决方案：**
- 分析工具依赖关系
- 并行执行无依赖的工具
- 使用 asyncio 实现异步调用

### 5. 缓存机制

**技术难点：** 减少重复查询，提升响应速度

**解决方案：**
- 基于内存的缓存实现
- TTL（生存时间）管理
- 缓存键设计：`tool_name:params_hash`

## 项目难点与解决方案

### 难点 1：复杂的依赖关系管理

**问题描述：** 工具之间存在复杂的依赖关系，需要正确排序执行

**解决方案：**
- 使用 DAG（有向无环图）表示依赖关系
- 拓扑排序确定执行顺序
- 参数占位符解析，支持前序步骤结果传递

### 难点 2：自然语言日期解析

**问题描述：** 用户使用各种自然语言表达日期，解析难度大

**解决方案：**
- 实现时间上下文管理器
- 支持相对日期和星期表达
- 时区处理（中国时区）
- 日期标准化

### 难点 3：数据源不稳定

**问题描述：** 第三方数据源可能不稳定或返回错误

**解决方案：**
- 实现多级降级策略
- 参数智能修复
- 错误检测和自动恢复
- 友好的错误提示

### 难点 4：性能监控和调试

**问题描述：** 复杂的执行流程难以监控和调试

**解决方案：**
- 实现企业级审计日志系统
- 结构化事件记录
- 执行链追踪
- 性能指标统计

### 难点 5：实时用户体验

**问题描述：** 复杂查询需要较长时间，用户体验差

**解决方案：**
- 实现流式响应（SSE）
- 实时推送执行状态
- 分步骤展示思考过程
- 异步工具调用
